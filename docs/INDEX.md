## ドキュメント管理ルール

### 更新タイミング

| ドキュメント | 更新タイミング |
|------------|--------------|
| 変更履歴.md | **すべての設定変更時に必須** |
| 開発フェーズ定義.md | フェーズ移行時、マイルストーン達成時 |
| 開発ステップ管理.md | ステップ完了時、進捗更新時 |
| README.md | 機能追加時、使い方変更時 |
| ARCHITECTURE.md | アーキテクチャ変更時 |

### ドキュメント命名規則

- **形式**: 日本語名（英語名）.md
- **例**: `変更履歴(CHANGELOG).md`
- **理由**: 日本語環境での可読性向上、英語環境との互換性維持

### ドキュメントレビュー

- **頻度**: 月次レビュー
- **対象**: すべてのドキュメント
- **目的**: 情報の正確性維持、古い情報の更新
