# 🚀 自動エラー検知・修復システム クイックスタート

> **所要時間**: 5分で理解、すぐに使える！

---

## 🎯 これは何？

あなたのプロジェクトを**24時間365日自動監視**し、エラーを見つけたら**自動で修復**するシステムです。

### できること

✅ 5分ごとに自動監視  
✅ エラーを自動検知  
✅ 可能なものは自動修復  
✅ 修復できないものはIssue作成  
✅ 詳細レポートを自動生成

---

## 🏃 すぐに使う（3ステップ）

### ステップ1: 確認

```bash
# 必要なファイルがあるか確認
ls .github/workflows/auto-error-fix-continuous.yml
ls scripts/health_monitor.py
ls scripts/auto_fix_daemon.py
```

すべてあれば OK！ ✅

### ステップ2: 手動実行してテスト

1. GitHubのリポジトリページを開く
2. **Actions** タブをクリック
3. **🤖 自動エラー検知・修復システム** を選択
4. **Run workflow** をクリック
5. 実行結果を確認

### ステップ3: 自動実行の確認

**何もしなくてOK！** システムは自動的に動作します。

- ⏰ **実行間隔**: 5分ごと
- 🎯 **実行条件**: エラーが検出されたとき
- 📍 **確認方法**: Actions タブで確認

---

## 📊 実行結果の見方

### ステータス一覧

| アイコン | ステータス | 意味 |
|---------|-----------|------|
| 🟢 | healthy | すべて正常 |
| 🟡 | degraded | エラーあるが修復中 |
| 🔴 | critical | 深刻なエラー |

### 確認場所

```
GitHub → Actions → 実行を選択 → Summary を確認
```

---

## ⚙️ 設定（オプション）

手動実行時に設定可能：

| 項目 | デフォルト | 説明 |
|------|----------|------|
| 最大ループ回数 | 15 | 修復を試みる最大回数 |
| Issue作成有無 | yes | エラー時にIssue作成 |
| ドライラン | no | 検知のみで修復しない |

---

## 🆘 困ったときは

### 実行されない

→ 正常です。エラーがない限り実行されません。

### エラーが修復されない

1. 最新のIssueを確認
2. エラー内容を把握
3. 手動で修復
4. Issueをクローズ
